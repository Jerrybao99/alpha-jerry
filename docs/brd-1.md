## 1. 执行摘要

`alpha-jerry` 是一套**面向 A 股基本面分析的 AI Native 工具集**，覆盖"数据采集 → 个股评分 → 个股评级 → 报告输出 → 持仓监控 → 热点追踪 → 推送通知"完整业务闭环，帮助个人投资者以低成本、可解释、可审计的方式识别优质公司并持续监控持仓风险。

产品形态为 Win/Mac 双端桌面应用，内置多 Agent 编排与本地知识库，用户可通过自然语言对话驱动全流程，数据全部本地存储以保护隐私。默认采用 DeepSeek 作为大模型，Tushare 作为 A 股数据源，遵循"成熟、开源、低成本"选型原则。

业务定位一句话：**把机构级的基本面筛选逻辑，做成个人投资者桌面上随时可对话的智能助手。**

## 2. 背景与问题陈述

### 2.1 业务背景

A 股市场上市公司超过 5000 家，个人投资者面临三重困境：

1. **信息过载**：财报、公告、新闻、热点海量且分散，人工难以全量覆盖。
2. **分析门槛高**：基本面分析需要财务三表、行业对照、估值模型等多维知识，小白难以独立完成。
3. **持仓监控被动**：买入后缺乏持续跟踪机制，常因风险事件（造假、质押、行业毁灭）滞后发现而受损。

### 2.2 问题陈述

| 编号 | 问题 | 当前痛点 | 期望状态 |
|---|---|---|---|
| BP-01 | 全市场筛选难 | 手动逐只看财报，效率极低 | 一键采集全 A 股并按统一规则评分 |
| BP-02 | 评判标准不一 | 凭感觉选股，结论不可复现 | 固化三维评分 + 行业权重 + 一票否决的可审计规则 |
| BP-03 | 风险识别滞后 | 造假/质押/行业颠覆发现太晚 | 一票否决前置剔除，持仓每日重算并高亮风险 |
| BP-04 | 热点与机会脱节 | 看到热点不知受益谁 | 热搜自动映射受益行业与个股 |
| BP-05 | 交互门槛高 | 专业工具学习成本高 | 自然语言对话即可驱动全部能力 |

### 2.3 市场参考

项目借鉴同类开源项目（OpenBB、daily_stock_analysis、Vibe-Trading、TradingAgents-CN）的工程实践与 Agent 架构，但聚焦**A 股基本面**这一垂直场景，并以**本地优先 + 对话驱动**作为差异化定位。

## 5. 业务范围

### 5.1 In Scope（首版必做）

1. 全 A 股基本面数据采集与特征工程落地。
2. 一票否决 + 三维评分（成长/稳健/回报）+ 行业权重 + 综合分。
3. 个股评级（皇冠明珠/优秀白马/鸡肋·观察/垃圾）。
4. 荐股 Top20 报告与持仓表生成（含 AI 亮点与点评）。
5. 热点追踪（每日 09:00/17:00）。
6. 持股分析（每日 09:00/17:00 重算、风险高亮、趋势对比、操作建议）。
7. 邮件推送（微信可选）。
8. 多 Agent 编排 + RAG 知识库 + 记忆 + 监控 + 反馈。
9. Win/Mac 双端桌面应用，自然语言对话交互。
10. 数据本地存储，定时自动更新。

### 5.2 Out of Scope（明确不做）

- 量化高频交易与盘中实时下单执行。
- 港股、美股等多市场（架构预留扩展位）。
- 云端 SaaS、多租户、多人协作。
- 纯技术面短线策略（本项目聚焦基本面）。
- 自动代客理财与资金托管。

### 5.3 假设

- A1：用户已自行办理 Tushare Pro 账户并取得可用 token 与积分。
- A2：用户已自行取得 DeepSeek API Key。
- A3：用户设备可访问 Tushare 与 DeepSeek 接口（网络可达）。
- A4：A 股财报口径在项目周期内无破坏性变更；若有，通过字段契约文档与测试同步。
- A5：用户投资决策仍由人完成，本工具仅提供分析辅助。

### 5.4 依赖

- D1：Tushare Pro 接口与积分。
- D2：DeepSeek LLM 推理服务。
- D3：百度/微博/东财等热搜来源的可访问性。
- D4：Win/Mac 打包工具链（Electron 等）。

## 6. 业务流程概述

业务主流程为四步核心流程 + 三项持续运营功能，均以本地数据文件为流转介质（沿用 BRD 原始约定，目录统一为 `data/`）。

### 6.1 核心流程（step 1 ~ step 4）

```
全 A 股清单
    │  ① 数据采集（Tushare，特征工程字段）
    ▼
data/fin/YYMMDD.xlsx
    │  ② 个股评分（一票否决 → 三维评分 → 行业权重 → 综合分）
    ▼
data/fin/YYMMDD-评分.xlsx
    │  ③ 个股评级（综合分 → 四级评级 + AI 点评）
    ▼
data/fin/YYMMDD-评级.xlsx
    │  ④ 报告输出（综合分降序 Top20 + 持仓表）
    ▼
data/analysis/YYMMDD-荐股.xlsx   ← 用户据此决策并录入持仓
data/hold/YYMMDD.xlsx        ← 持仓基线
```

### 6.2 持续运营流程

```
每日 09:00 / 17:00
    ├── 热点追踪：采热搜 → LLM 识别机会 → 受益行业/个股 → data/hot/
    └── 持股分析：重爬重算持仓 → 风险高亮 → 趋势对比 → 操作建议
            → data/hold/YYMMDD-09.xlsx / -17.xlsx
    │
    ▼
推送（邮件完整 HTML / 微信摘要）
```

### 6.3 更新流程

- 时间触发，自动执行。
- 每季度财报季后更新基本面数据。
- 每月更新资金面数据。

## 7. 业务规则（业务资产，不可漂移）

> 以下规则是投资方法论的业务沉淀，属项目核心资产。PRD 与代码实现必须与其逐条对齐，并以纯函数单测验证对齐（详见 PRD §6）。

### 7.1 一票否决规则

| 否决项 | 判定条件 |
|---|---|
| 造假嫌疑 | 货币资金高但利息收入极低；经营现金流连续多年远低于净利润 |
| 行业毁灭 | 技术路线被颠覆（如胶卷、燃油车零部件） |
| 诚信问题 | 频繁更换审计机构（3 年换 2 次以上）；大股东质押率 >70%；曾因信披违规被立案 |

触发任一即剔除出评级，但需记录可审计。

### 7.2 三维评分（各 1-10 分）

#### 7.2.1 成长性得分

| 分数 | 营收增速 | 净利增速 | 盈利质量 | 现金流匹配 |
|---|---|---|---|---|
| 9-10 | > 40% | >50% | 毛利上升 | >1.2 |
| 7-8 | 25-40% | 30-50% | 毛利稳定 | 1.0-1.2 |
| 5-6 | 15-25% | 15-30% | 毛利微降 | 0.8-1.0 |
| 3-4 | 5-15% | 0-15% | 毛利显著降 | 0.5-0.8 |
| 1-2 | < 5%或负 | 负增长 | 毛利崩盘 | <0.5 |

#### 7.2.2 稳健性得分

| 分数 | 资产负债率 | 流动比率 | 存货周转率 | 审计意见 |
|---|---|---|---|---|
| 9-10 | < 40% | > 2.5 | 快于平均 | 无风险 |
| 7-8 | 40-60% | 1.5-2.5 | 持平平均 | 结构清晰 |
| 5-6 | 60-75% | 1.0-1.5 | 略慢平均 | 少量关联交易 |
| 3-4 | 75-85% | < 1.0 | 积压严重 | 审计非标 |
| 1-2 | > 85% | 断裂边缘 | 卖不出去 | 存贷双高 |

#### 7.2.3 资金回报得分

| 分数 | ROE | 自由现金流 | 分红率 | 估值 |
|---|---|---|---|---|
| 9-10 | > 20% | 持续正且增长 | 分红 > 40% | PEG < 1 |
| 7-8 | 15-20% | 正偶尔波动 | 分红 30-40% | PEG 1-1.5 |
| 5-6 | 10-15% | 接近 0 | 象征性分红 | PEG > 1.5 |
| 3-4 | 6-10% | 为负 | 不分红 | 历史高位 |
| 1-2 | < 6% | 巨额负值 | 铁公鸡 | 泡沫化 |

### 7.3 行业权重对照表

| 成长性 | 30% | 双位数+预收增 | 营收10-20% | 3-10% | 负增长 |

> 权重以区间给出时，业务默认取区间中点；评审后可冻结为确定值。

### 7.4 综合分公式

$$
\text{综合分} = \text{成长分} \times \text{成长权重} + \text{稳健分} \times \text{稳健权重} + \text{回报分} \times \text{回报权重}
$$

保留 1 位小数。
